# Aufgabe 08 — Spickzettel: Product Owner Gespraech

## Iron Triangle (immer im Kopf behalten)

```
         QUALITAET
            /\
           /  \
          /    \
    ZEIT /______\ SCOPE

Qualitaet = nicht verhandelbar (E-Health!)
Zeit      = vom PO vorgegeben
Scope     = das worüber wir verhandeln → MVP!
```

---

## Die 5 Schluesselwoerter fuer das Interview

**MVP** — Minimum Viable Product
→ Was ist das Minimum das echten Wert liefert?

**INVEST** — Kriterien fuer gute User Stories
→ Independent, Negotiable, Valuable, Estimable, Small, Testable

**Vertical Slicing** — Stories schneiden
→ Nicht nach Technik-Schichten, sondern nach Business-Wert

**Technical Debt** — immer als Business-Risiko erklaeren
→ Nicht "Code ist haesslich" → "Sicherheitszertifikat laeuft ab"

**Gherkin** — Given / When / Then
→ Akzeptanzkriterien die PO und Entwickler beide verstehen

---

## Schnell-Antworten

| Frage | Kern der Antwort |
|---|---|
| Enge Deadline? | Iron Triangle → Scope reduzieren → MVP vorschlagen |
| Vage Anforderung? | INVEST + Vertical Slicing + Gherkin |
| Technical Debt? | In Geschaeftsrisiken uebersetzen → 15% Budget |
| Unsicherer Workaround? | Job-to-be-Done → sichere Alternative anbieten |
| Epic vs Story? | Epic = zu gross, Story = 1 Sprint, testbar |
